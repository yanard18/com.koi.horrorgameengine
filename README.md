### Requirements
- Unity 6000.3.6f1
- Compatibility With Unity 2022.3.62f3 LTS
- Server/Client architecture with both singleplayer & multiplayer support
- Custom editors for tools and artist first design principle
- Each mechanic should be configurable with its all parameters
- Prioritize FPS mechanics and later add TPS (note that competitor horror kit only support fps)
- Design as custom unity package: https://docs.unity3d.com/2020.1/Documentation/Manual/CustomPackages.html
- Two demo maps(games) should be implemented before publish. One can be "slender the eight pages" clone.
- Unit Tested Development
- Online documentation

### User Requirements
- Basic FPS movement (Needs to be detailed more before starting)
- Jump mechanic
- Hiding mechanic (hiding inside a locker etc)
- Procedural head bob (probably better if LLM code this)
- Sound Engine (FMOD is the best soung engine like asset, if inspiration needed)
- Basic Inventory (hard to implement in multiplayer)
- Tetris Inventory (harder to implement in multiplayer)
- Flashlight mechanic & Battery (integrate with inventory systems)
- Object Inspect
- Puzzles (integrate with object inspection)
- Quest System
- Guns, melee weapons, etc...
- Narrative Mechanic
- NPC management (enforce developers to spawn entities with our framework to provide useful handles to actors)
- Trigger Boxes
- Door Open system and physics
- User Designed Custom Horror NPCs (bunch of different character can be inspired by "Lethal Company")
